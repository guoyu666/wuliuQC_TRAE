# 物流QC框子收发记录小程序 - 规格说明书

## 1. 项目概述

- **项目名称**: 物流QC框子收发记录
- **项目类型**: 微信小程序
- **核心功能**: 记录库房每天蓝色框子和红色框子的发出和收回数量
- **目标用户**: 库房管理人员

## 2. 功能列表

### 2.1 记录功能
- 记录蓝色框子发出数量
- 记录蓝色框子收回数量
- 记录红色框子发出数量
- 记录红色框子收回数量
- 支持当天多次记录，自动累加

### 2.2 数据展示
- 今日收发汇总（当前日期的发出和收回总数）
- 历史记录列表（按日期分组显示）

### 2.3 数据管理
- 本地数据持久化存储
- 支持查看历史每天的收发数据

## 3. UI/UX 设计方向

### 3.1 视觉风格
- 简洁实用的 Material Design 风格
- 蓝色为主色调（对应蓝色框子）
- 红色为辅助色（对应红色框子）

### 3.2 色彩方案
- 主色: #1976D2 (蓝色)
- 辅助色: #D32F2F (红色)
- 背景色: #F5F5F5
- 文字色: #333333
- 次要文字: #666666

### 3.3 布局方案
- **首页**: 顶部显示今日日期和汇总数据，中间为输入区域，底部为快捷操作按钮
- **历史记录页**: 列表形式展示每日数据，支持下拉刷新

### 3.4 交互设计
- 数字输入使用减号和加号按钮
- 记录按钮使用明显的确认操作
- 操作完成后有Toast提示

## 4. 数据结构

### 记录项 (Record)
```json
{
  "id": "记录ID",
  "date": "2024-01-01",
  "blueOut": 10,
  "blueIn": 5,
  "redOut": 8,
  "redIn": 3,
  "createTime": "2024-01-01 10:30:00"
}
```

## 5. 页面结构

1. **首页 (index)**
   - 今日日期显示
   - 今日汇总卡片（蓝色发出/收回、红色发出/收回）
   - 记录输入区域
   - 提交记录按钮
   - 跳转历史记录入口

2. **历史记录页 (history)**
   - 按日期倒序排列的记录列表
   - 每日数据汇总显示
