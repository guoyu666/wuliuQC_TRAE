# 物流QC框子收发记录小程序 - 规格说明书

## 1. 项目概述

- **项目名称**: 物流QC框子收发记录
- **项目类型**: 微信小程序
- **核心功能**: 记录库房每天蓝色框子和红色框子的发出和收回数量，包含线路和车辆信息
- **目标用户**: 库房管理人员、物流调度员

## 2. 技术架构

### 2.1 云开发架构
- **后端**: 微信云开发
- **数据库**: 云开发数据库（自动创建 users 和 records 集合）
- **云函数**: login（自动登录）、syncRecords（数据同步）
- **本地存储**: wx.setStorageSync（混合模式，数据同时存储本地和云端）

### 2.2 数据同步策略
- **写入**: 本地优先，云端异步同步
- **读取**: 云端优先，云端获取失败时使用本地
- **合并**: 多设备数据通过 syncRecords 云函数合并

## 3. 功能列表

### 3.1 记录功能
- 记录线路名称（如：京沪线、沪宁线）
- 记录运送车辆车牌号
- 记录物流中心蓝色框子发出数量
- 记录物流中心红色框子发出数量
- 记录蓝色框子发出数量
- 记录蓝色框子收回数量
- 记录红色框子发出数量
- 记录红色框子收回数量
- 支持当天多次记录，每条记录独立保存
- 每条记录包含独立的线路、车牌、物流发出和收发数量

### 3.2 数据展示
- 今日收发汇总（当前日期的发出和收回总数）
- 当日记录条数统计
- 数据可视化柱状图
- 历史记录列表（按日期分组显示）

### 3.3 快捷操作
- 昨天快捷跳转
- 上周快捷跳转
- 月初快捷跳转

### 3.4 数据管理
- 本地数据持久化存储
- 云端数据同步（自动）
- 支持查看历史每天的收发数据
- 数据导出功能（包含线路、车牌信息）
- 编辑、删除、添加备注功能

## 4. UI/UX 设计方向

### 4.1 视觉风格
- 简洁实用的 Material Design 风格
- 蓝色为主色调（对应蓝色框子）
- 红色为辅助色（对应红色框子）

### 4.2 色彩方案
- 主色: #1976D2 (蓝色)
- 辅助色: #D32F2F (红色)
- 背景色: #F5F5F5
- 文字色: #333333
- 次要文字: #666666

### 4.3 布局方案
- **首页**: 日期选择器、快捷日期按钮、当日汇总卡片、数据可视化图表、新增记录表单
- **历史记录页**: 列表形式展示每日数据，支持分页、导出、编辑、删除
- **统计报表页**: 月度/年度汇总、收发对比图、趋势图

### 4.4 交互设计
- 文本输入使用标准输入框
- 数字输入使用减号和加号按钮

- 操作完成后有Toast提示
- 日期切换带动画效果

## 7. 云函数列表

### 7.1 login 云函数
- **功能**: 自动登录，获取/创建用户记录
- **返回**: openid、是否新用户、用户信息

### 7.2 syncRecords 云函数
- **功能**: 数据同步、合并、清理
- **支持操作**:
  - `upload`: 上传本地记录到云端
  - `download`: 从云端拉取记录
  - `merge`: 合并本地和云端数据
  - `clear`: 清除云端数据

## 8. 数据库集合

### 8.1 users 集合
- 存储用户基本信息
- 按 openid 索引

### 8.2 records 集合
- 存储所有收发记录
- 按 openid 和 createTime 索引
